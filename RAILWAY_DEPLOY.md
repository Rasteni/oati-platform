# Деплой на Railway

Эта инструкция проведёт через всё: с момента "у меня есть папка с проектом" до "вот публичный URL, любой может открыть".

Время: 15-20 минут на первый раз.

## Что получишь

- Публичный HTTPS-URL вида `https://oati-geo-production.up.railway.app`
- Постоянно работающий сервис (не засыпает в отличие от Render free)
- Данные сохраняются между перезапусками (persistent volume)
- Автодеплой: запушил в GitHub → Railway сам пересобрал и обновил

## Стоимость

Railway даёт **$5 кредитов в месяц бесплатно**. Этого хватает на маленький проект как у нас (примерно $3-4/мес расход). Платёжная карта **не требуется** на старте — пока укладываешься в $5, можешь не привязывать.

Если кредитов перестанет хватать — привязываешь карту и платишь только за overage (обычно $1-2 сверху).

## Шаг 1. Залить проект на GitHub

Если GitHub-аккаунта нет — заведи на [github.com](https://github.com), это бесплатно.

### Вариант А: через веб-интерфейс (проще)

1. На [github.com](https://github.com) → правый верхний угол → **+** → **New repository**
2. **Repository name**: `oati-geo` (или любое)
3. **Public** или **Private** — на твой выбор (Railway работает с обоими)
4. **НЕ** ставь галочки «Add a README», «.gitignore», «license» — у нас они уже свои
5. Жми **Create repository**
6. На странице нового пустого репо нажми **uploading an existing file**
7. Перетащи **содержимое** папки `oati-lite` (не саму папку, а файлы внутри: `backend/`, `frontend/`, `Dockerfile`, `docker-compose.yml`, `README.md`, `railway.toml`, `.gitignore`, `.dockerignore`, `run.py`, `run.bat`, `run.sh`, `load_real_districts.py`)
8. Внизу кнопка **Commit changes**

### Вариант Б: через git CLI (если умеешь)

```bash
cd oati-lite
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ТВОЙ_НИК/oati-geo.git
git push -u origin main
```

## Шаг 2. Регистрация на Railway

1. Открой [railway.app](https://railway.app)
2. **Login** → **Login with GitHub**
3. Разреши Railway доступ к твоему GitHub
4. Готово, ты внутри

При первом входе Railway может попросить пройти короткий онбординг — пропусти.

---

## Шаг 3. Создание проекта из GitHub-репозитория

1. На главной Railway → **+ New Project**
2. Выбери **Deploy from GitHub repo**
3. Если репозиторий не показывается — нажми **Configure GitHub App**, разреши доступ к нужному репо
4. Найди `oati-geo` в списке → клик на него
5. Railway начнёт сборку автоматически

Что происходит сейчас под капотом:
- Railway клонирует репозиторий
- Видит `Dockerfile` и `railway.toml`
- Собирает Docker-образ
- Запускает контейнер

На страничке сервиса увидишь логи в реальном времени. Сборка занимает 3-5 минут.

---

## Шаг 4. Сгенерировать публичный URL

1. Когда сборка закончится (зелёный значок «Active»), кликни на сервис
2. Вкладка **Settings**
3. Раздел **Networking** → **Generate Domain**
4. Railway создаст URL вида `oati-geo-production-XXXX.up.railway.app`
5. Открой его в браузере — должна появиться карта

🎉 Сервис работает. Теперь по этому URL заходит кто угодно.

---

## Шаг 5. Persistent storage (чтобы БД не пропала)

По умолчанию контейнер Railway эфемерный — после рестарта (например, при новом деплое) файлы пропадают. БД нужно сохранить на постоянный диск.

1. На странице сервиса → вкладка **Settings**
2. Раздел **Volumes** → **Add Volume**
3. Параметры:
   - **Mount path**: `/app/data`
   - **Size**: `1 GB` (хватит на миллион объектов)
4. Жми **Add**
5. Railway перезапустит сервис с диском

Теперь `oati.db` сохраняется в этот volume и переживает рестарты/редеплои.

### Второй volume для фото

Те же шаги, но **Mount path** = `/app/uploads`, размер 1-5 GB в зависимости от ожидаемого объёма фото.

## Шаг 6. (опционально) Свой домен

Если хочешь URL покрасивее:

1. Купи домен где-нибудь (Reg.ru, Beget, Namecheap) — `.ru` ~150 руб/год, `.tech`/`.app` ~$15/год
2. На странице сервиса Railway → **Settings** → **Networking** → **Custom Domain**
3. Введи свой домен, например `oati-geo.ru`
4. Railway покажет, какие DNS-записи (CNAME) нужно добавить у регистратора
5. Заходишь в личный кабинет регистратора → DNS → добавляешь CNAME запись
6. Через 5-30 минут (DNS-пропагация) домен заработает с HTTPS

## Автодеплой при изменениях

Это работает из коробки. Когда ты:
1. Меняешь файлы локально
2. Заливаешь на GitHub (commit + push, или через веб-интерфейс)

Railway автоматически:
1. Видит новый коммит
2. Пересобирает образ
3. Деплоит новую версию
4. Старая отключается без даунтайма

Удобно для итераций: правишь CSS, заливаешь на GitHub, через 2 минуты обновления видны на проде.

## Загрузка реальных границ Москвы (после деплоя)

Один раз после первого запуска нужно загрузить точные границы из OSM:

1. На странице сервиса Railway → вкладка **Settings** → раздел **Service**
2. Найди кнопку **... (три точки)** или меню сервиса
3. Открой встроенный терминал (если есть кнопка **Terminal** или **Shell**)
4. Если терминала нет — можешь временно поменять start command:
   - **Settings** → **Deploy** → **Custom Start Command**
   - Поменяй на: `python -m app.scripts.load_real_districts && uvicorn app.main:app --host 0.0.0.0 --port $PORT`
   - Сохрани, дождись пересборки
   - Затем верни обратно: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`

Альтернативно — добавить эндпоинт в API для запуска скрипта по нажатию кнопки. Скажи, если нужно, добавлю.

## Мониторинг расхода

Railway показывает потребление прямо на дашборде:
- **Metrics** на странице сервиса — CPU, RAM, Network в реальном времени
- **Usage** в настройках аккаунта — сколько $ из $5 уже потрачено в этом месяце

Маленький проект как наш потребляет ~$3-4/мес. Если выйдешь за $5 без привязанной карты — сервис временно остановится до начала следующего месяца.

## Возможные проблемы

### Сборка падает на `pip install pandas`
Pandas компилируется долго и требует памяти. Railway иногда отваливается по таймауту. Решение: в `requirements.txt` зафиксировать колёса (wheels) — уже сделано в текущем `requirements.txt`.

### Healthcheck не проходит
Проверь, что `/api/health` доступен (он есть в `backend/app/main.py`). Если поменял путь — обнови `healthcheckPath` в `railway.toml`.

### Сервис стартует, но карта не видна
Открой логи на Railway — если видишь ошибки про PostGIS или что-то ещё — пиши, разберёмся.

### Хочу полностью обнулить и начать заново
**Settings** → прокрутить вниз → **Danger** → **Delete Service**. Потом заново через **+ New** → **Deploy from GitHub repo**.

## Безопасность

Сейчас сервис **полностью открыт** — кто угодно может зайти, загрузить данные, очистить базу. Для демо/портфолио это нормально, но если планируешь использовать всерьёз:

1. Включить простой Basic Auth перед всеми роутами (могу добавить, ~20 строк кода)
2. Или добавить настоящую авторизацию с логинами (см. ветку в README про fastapi-users)
3. Ограничить CORS — в `backend/app/config.py` поменять `CORS_ORIGINS = ["*"]` на конкретный домен

Скажи, нужно ли — закрою.
