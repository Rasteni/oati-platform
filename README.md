# ОАТИ · Геоаналитика

Веб-приложение для пространственной аналитики объектов контроля ОАТИ. Карта Москвы, импорт CSV/Excel, фильтры, тайм-слайдер, дашборд, прогноз, загрузка открытых данных Москвы, геокодинг адресов.



## Быстрый старт (локально)

Самый простой способ — Docker. Поставил Docker Desktop один раз, потом весь проект — две команды.

```bash
docker compose up --build
```

Откроется на `http://localhost:8000`. Первый запуск 2-3 минуты (скачивается образ Python и ставятся зависимости), дальше — 5-10 секунд.

Остановить:
```bash
docker compose down
```

База данных и фото сохраняются в папках `data/` и `uploads/` рядом с проектом. После `docker compose down` данные остаются — следующий запуск продолжит с того же места.

## Установка Docker Desktop

### Windows

1. Скачать установщик: [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/) → кнопка «Download for Windows»
2. Запустить `Docker Desktop Installer.exe`, пройти установку (могут запросить включить WSL2 — соглашайся)
3. **Перезагрузить компьютер**
4. Запустить Docker Desktop из меню «Пуск» — должна появиться иконка кита в трее. Дождаться, пока она перестанет двигаться (Docker запускается)
5. Открыть **PowerShell** (правый клик на «Пуск» → Terminal или Windows PowerShell) и проверить:
   ```
   docker --version
   ```
   Должно показать что-то вроде `Docker version 27.x.x`

### macOS

1. Скачать установщик: [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/) → кнопка «Download for Mac»
2. Важно: выбрать **Apple Silicon** (если M1/M2/M3) или **Intel chip** (если старый Mac на Intel). Если не знаешь — открой меню Apple → «Об этом Mac», там написано «Чип»
3. Открыть скачанный `.dmg`, перетащить Docker в Applications
4. Запустить Docker.app из Applications. Согласиться на права администратора при первом запуске
5. Дождаться, пока иконка кита в верхней панели перестанет анимироваться
6. Открыть **Терминал** (Cmd+Space → «Terminal») и проверить:
   ```
   docker --version
   ```

### Ubuntu / Linux

Удобнее ставить через официальный репозиторий — Docker Desktop на Linux есть, но Docker Engine лучше:

```bash
# Удалить старые версии (если есть)
sudo apt remove docker docker-engine docker.io containerd runc

# Установить
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Добавить себя в группу docker, чтобы не писать sudo каждый раз
sudo usermod -aG docker $USER
newgrp docker

# Проверить
docker --version
docker compose version
```

После добавления в группу — **перезайди в систему** (или открой новый терминал).

## Запуск проекта

### Все ОС, одинаково:

1. Скачать `oati-lite.zip` и распаковать
2. Открыть терминал/PowerShell **внутри папки oati-lite**
   - **Windows**: открой папку в проводнике → правый клик в пустом месте → «Открыть в Terminal» (или Shift + правый клик → «Открыть здесь окно PowerShell»)
   - **macOS**: открой Terminal → набери `cd ` (с пробелом!) → перетащи папку oati-lite в окно терминала → Enter
   - **Linux**: правый клик на папке → «Open in Terminal»
3. Выполнить:
   ```
   docker compose up --build
   ```
4. Открыть в браузере `http://localhost:8000` (если автоматически не открылось)
5. Нажать «+500 демо» в боковой панели — увидишь карту с объектами

Сервер пишет логи в терминал. Чтобы остановить — `Ctrl+C` в терминале или в новом окне:
```
docker compose down
```

### Второй и последующие запуски

```
docker compose up
```

Без `--build` — пересборка не нужна, образ уже собран. Запускается за 5 секунд.

## Загрузка реальных границ Москвы

После первого запуска можно загрузить точные границы 12 округов из OpenStreetMap:

```bash
docker compose exec app python -m app.scripts.load_real_districts
```

Скрипт обращается к Overpass API (~30-60 секунд), сохраняет полигоны в БД, пересчитывает привязку объектов. После завершения — перезагрузи страницу.

## Где хранятся данные

- `data/oati.db` — SQLite-база с объектами, проверками, округами
- `uploads/` — загруженные фотографии объектов

Можно копировать `data/oati.db` себе на флешку — это полный backup. Чтобы перенести проект на другой компьютер — копируй всю папку oati-lite вместе с `data/` и `uploads/`.

## Возможности

### Карта
- Точки объектов, кластеры, теплокарта, хороплет по округам
- 5 подложек на выбор: CARTO Voyager, OpenStreetMap, Positron, Humanitarian, Спутник
- Кликабельные округа Москвы
- Тайм-слайдер для анимации по месяцам

### Фильтры
- По статусу (норма / замечание / нарушение / критическое / на проверке)
- По округу (множественный выбор)
- По типу объекта
- По периоду последней проверки

### Импорт данных
- CSV / Excel — drag-n-drop или клик на зону загрузки
- Геокодинг адресов через Nominatim — если в файле есть адреса, но нет координат
- Загрузка из data.mos.ru: дворовые территории, контейнерные площадки, дома, ордера ОАТИ

### Дашборд
- KPI: всего объектов, норма, замечания, нарушения
- Графики: статусы, топ округов по нарушениям, типы объектов
- Динамика проверок помесячно со стеком статусов
- Прогноз на 6 месяцев (тренд + сезонность)
- Сводная таблица округов с долей нарушений
- Производительность инспекторов: проверок, объектов, доля нарушений, средний интервал

### Экспорт
- CSV и Excel — выгрузка отфильтрованной выборки

## Подключение реальных карт (2ГИС, Яндекс)

В сайдбаре есть переключатель подложек. Из коробки доступны бесплатные:
- CARTO Voyager — светлая дашбордная (по умолчанию)
- OpenStreetMap — самая детальная (улицы, дома, подъезды)
- Спутник Esri — спутниковые снимки

Для **2ГИС**: получить ключ на [dev.2gis.ru](https://dev.2gis.ru), в `frontend/app.js` найти `BASE_LAYERS`, раскомментировать блок `2gis`.

Для **Яндекс Карт**: нужен плагин Leaflet.Yandex или нативный Yandex Maps JS API. См. комментарии в `frontend/app.js`.

## Возможные проблемы

### Windows: «WSL 2 installation is incomplete»
Установи WSL2: открой PowerShell от администратора, выполни `wsl --install`, перезагрузи компьютер.

### Windows: иконка Docker не появляется или висит в «Starting»
Docker Desktop требует Hyper-V (Windows Pro) или WSL2 (Windows Home). Зайди в «Включение или отключение компонентов Windows», убедись что включён «Подсистема Windows для Linux» и «Платформа виртуальной машины».

### macOS на M1/M2/M3: ошибка про архитектуру
Убедись, что скачал именно Apple Silicon версию. Если случайно поставил Intel — снеси и поставь правильную.

### «Port 8000 is already in use»
Что-то уже занимает порт 8000. Найди и закрой — на Windows через `netstat -ano | findstr :8000`, на Mac/Linux через `lsof -i :8000`. Или поменяй `8000:8000` в docker-compose.yml на `8001:8000`, открывай по `localhost:8001`.

### Pip висит при сборке образа
Это нормально — pandas и numpy тяжёлые, сборка занимает 1-3 минуты. Не отменяй.

### Хочу полностью обнулить
```
docker compose down
rm -rf data/ uploads/
docker compose up --build
```

## Альтернатива без Docker

Если Docker не подходит — есть скрипт `run.py` для запуска через Python venv:

```bash
# Mac/Linux
python3 run.py

# Windows — двойной клик на run.bat
```

Подробнее см. в файлах `run.sh` / `run.bat`. Требует Python 3.10+.
